¿Qué es una Base de Datos? ¿Por qué no usar solo Excel?

Una base de datos es un sistema organizado para almacenar, relacionar y consultar grandes volúmenes de información de forma estructurada y sin duplicación.

La diferencia clave: Excel guarda datos en celdas individuales. Una base de datos guarda datos en tablas relacionadas que se conectan entre sí. Así, el nombre de un cliente se guarda UNA sola vez, no en cada fila de ventas.

Conceptos Fundamentales de Bases de Datos

Los 5 conceptos que debes dominar:

ConceptoQué esEjemplo
TablaConjunto de registros del mismo tipoTabla de Clientes
Campo (columna)Una característica de la tablaNombreCliente, Ciudad
Registro (fila)Un elemento de la tablaFila de un cliente específico
Clave primaria (PK)Identificador único por registroID_Cliente (nunca se repite)
Clave foránea (FK)Referencia a la PK de otra tablaID_Cliente en tabla Ventas
RelaciónVínculo entre dos tablas por las claves1 cliente → muchas ventas

Tipos de relaciones:

1 a muchos (la más común)

Un cliente puede tener muchas ventas. Una venta pertenece a un solo cliente.

Clientes (1) ──── Ventas (∞)

Muchos a muchos

Un pedido puede tener muchos productos. Un producto puede estar en muchos pedidos.

Pedidos (∞) ──── Productos (∞)

Requiere tabla intermedia: Detalles_Pedido

Microsoft Access — Crear Tablas y Relaciones

Crear una tabla en Access:

  • Archivo nuevo → Base de datos en blanco
  • Pestaña "Crear" → "Diseño de tabla"
  • Define cada campo: nombre, tipo de dato, tamaño
  • Clic derecho en el campo ID → "Clave principal"
  • Guardar con nombre (ej: "tblClientes")

Tipos de datos Access:

TipoPara qué
Texto cortoNombres, ciudades, códigos (máx 255)
NúmeroCantidades, IDs numéricos
MonedaPrecios, salarios (2 decimales)
Fecha/HoraFechas de pedido, registro
Sí/NoEstados booleanos (activo/inactivo)
AutonumeraciónID automático (1, 2, 3...)

Crear Relaciones:

  • Pestaña "Herramientas de base de datos" → "Relaciones"
  • Agrega las tres tablas al panel
  • Arrastra ID_Cliente de CLIENTES hacia ID_Cliente en PEDIDOS
  • En el cuadro: activa "Exigir integridad referencial"
  • Clic "Crear" → aparece la línea de relación
  • Repite para PRODUCTOS y PEDIDOS
Integridad referencial: Actívala siempre. Impide que se guarde un pedido con un ID_Cliente que no existe en la tabla CLIENTES. Protege la calidad de los datos.

Consultas en Access — Filtrar y Analizar Datos

Consultas visuales (QBE):

Access permite crear consultas con una interfaz visual: arrastras los campos que necesitas, defines los criterios, y Access genera el SQL.

  • Pestaña "Crear" → "Diseño de consulta"
  • Agrega las tablas necesarias
  • Arrastra los campos al cuadrícula inferior
  • En "Criterios" escribe la condición
  • Clic en "Ejecutar" (botón !)
Criterios útiles:
"Barranquilla" — exacto
Like "Bar*" — empieza con
>1000000 — mayor que
Between #1/1/2025# And #3/31/2025# — rango de fechas

SQL básico (lo que genera Access):

-- Pedidos de clientes de Barranquilla
SELECT
  C.NombreCliente,
  P.FechaPedido,
  PR.NombreProducto,
  P.Cantidad
FROM CLIENTES AS C
INNER JOIN PEDIDOS AS P
  ON C.ID_Cliente = P.ID_Cliente
INNER JOIN PRODUCTOS AS PR
  ON PR.ID_Producto = P.ID_Producto
WHERE C.Ciudad = 'Barranquilla'
ORDER BY P.FechaPedido DESC;

Formularios e Informes en Access

Formularios — Entrada de Datos Fácil:

Los formularios permiten ingresar y editar registros de forma ordenada, sin ver la tabla completa.

  • Pestaña "Crear" → "Asistente para formularios"
  • Selecciona la tabla (ej: CLIENTES)
  • Elige los campos que quieres mostrar
  • Elige el diseño (columnas, tabular)
  • Accede: el formulario muestra un cliente a la vez
Ventaja: El usuario solo puede ingresar los campos correctos. No puede accidentalmente borrar datos de otra fila o columna.

Informes — Reportes Listos para Imprimir:

Los informes en Access generan reportes formateados listos para impresión o PDF.

  • Pestaña "Crear" → "Asistente para informes"
  • Selecciona la consulta base
  • Define agrupaciones (ej: por Ciudad)
  • Define ordenamiento y subtotales
  • Elige diseño → el informe se genera
Caso real: El auxiliar de logística genera cada lunes un informe de pedidos pendientes de Barranquilla, directamente desde Access, con un clic. Sin copiar y pegar de Excel.
